# Not Recommended: Abstract-Only Method Description

Do not describe the proposed method only with abstract benefits.

Weak pattern:

```text
To solve the problem, we propose a novel framework that effectively captures user preference and improves recommendation quality.
The framework is simple, general, and efficient.
```

Why this is weak:

1. It does not say what the framework does.
2. It does not connect a design to the stated limitation.
3. It creates unsupported praise words.

Better pattern:

```text
To solve the problem, we propose [Name], which [specific mechanism].
Given [input], [Name] first [step 1], then [step 2], and finally [output].
This design addresses [limitation] because [reason].
```
