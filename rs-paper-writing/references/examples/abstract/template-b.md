# Abstract Template B: Diagnosis to Method

Use for LLM-based recommendation, fairness, debiasing, causal recommendation, or objective-level failures.

```text
LLM-based recommendation has recently reformulated personalized recommendation as conditional generation over item titles or identifiers.
However, we find that [existing objective/framework] amplifies [bias/failure] because [technical reason].
This issue leads to [recommendation-specific consequence], especially for [affected users/items/settings].
To mitigate this problem, we propose [Method Name], a [framework/objective] that [core idea].
[Method Name] [main module 1] and [main module 2], enabling the recommender to [benefit].
Experiments show that [Method Name] improves [utility metric] while reducing [bias/fairness/validity metric] over [baseline group].
```

Reusable logic:

1. Introduce the paradigm.
2. Diagnose the failure.
3. Explain why the failure matters.
4. Propose the solution.
5. Report both primary and secondary evidence.
